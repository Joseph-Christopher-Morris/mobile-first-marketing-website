# Desktop Optimisation Instructions

## 1. Desktop Priorities
- More space allows deeper explanations.
- Maintain clarity and prevent text bloat.
- Reinforce proof and authority elements.

## 2. Desktop CTA Rules
- Main CTA: **Call for a Free Ad Plan**
- Sticky CTA: appears once user scrolls past hero.
- CTA must remain contextual to the service.

## 3. Desktop Layout Requirements
- Strong hierarchy: Hero → Proof → Services → Case Studies → CTA.
- Use multi-column layouts where helpful.
- Keep testimonials longer than mobile but still concise.

## 4. Desktop Copy Guidelines
- Speak directly to business owners.
- Each section must answer: What? Who for? Why? What next?
- Avoid fluff or abstract marketing terms.

## 5. Desktop Proof Elements
Include above the fold or immediately after:
- Press logos
- Case studies
- Relevant stats (e.g., 82% faster load times)

## 6. Desktop Speed Requirements
- Preload hero image.
- Compress all media assets.
- Ensure LCP under 2 seconds.

## 7. Desktop Content Hierarchy
1. Strong hero with benefit-focused H1  
2. Subheading that expands value  
3. Clear CTAs  
4. Proof  
5. Service breakdown  
6. Case studies  
7. CTA
